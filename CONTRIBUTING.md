# Contributing to Drawgo

Thanks for your interest in contributing! This project has two main codebases:

- **Go backend** (`backend/`) — REST API, WebSocket, auth, storage
- **Next.js frontend** (`src/`) — React UI with Excalidraw

## How Can I Contribute?

### Reporting Bugs

Before creating bug reports, check existing issues. Include:

- Clear title and reproduction steps
- Expected vs observed behavior
- Relevant logs (`docker compose logs backend` or browser console)
- Environment (OS, browser, Go/Node versions)

### Suggesting Features

Use a clear title, describe the feature, explain why it's useful, and list alternatives considered.

### Pull Requests

1. Fork the repo and create your branch from `main`
2. Follow the [Development Setup](#development-setup) below
3. Make your changes
4. Run tests (see [Testing](#testing))
5. Run linters (`pnpm lint` for frontend, `golangci-lint run` for backend)
6. Commit with a [conventional message](#commit-messages)
7. Push and open a Pull Request

## Development Setup

### Prerequisites

| Tool           | Version  | Purpose           |
| -------------- | -------- | ----------------- |
| Go             | 1.25+    | Backend           |
| Node.js        | 22+      | Frontend          |
| pnpm           | 10+      | Package manager   |
| Docker Compose | v2+      | PostgreSQL, MinIO |

### Getting Started

```bash
# 1. Clone and enter the project
git clone https://github.com/<your-fork>/drawgo.git
cd drawgo

# 2. Copy environment config
cp .env.example .env

# 3. Start infrastructure (PostgreSQL + MinIO)
docker compose up -d postgres minio

# 4. Run the Go backend (auto-runs migrations)
cd backend
go run ./cmd/server
# Backend runs on :8080

# 5. In another terminal — run the frontend
cd drawgo
pnpm install
pnpm dev
# Frontend runs on :3000
# In Docker, Caddy proxies /api/v1/* to the backend
```

### Full Stack via Docker

```bash
docker compose up --build
# Frontend: http://localhost:3021
# Backend:  http://localhost:8090
# MinIO:    http://localhost:9011
```

## Code Style

### Go Backend

- **Formatting:** `gofmt` / `goimports` — enforced by CI
- **Error handling:** Always wrap with context (`fmt.Errorf("save board: %w", err)`)
- **Naming:** Exported types PascalCase, unexported camelCase
- **Interfaces:** Accept interfaces, return structs
- **SQL:** Parameterized queries only (pgx `$1` placeholders)
- **Validation:** Use `go-playground/validator` struct tags

### Next.js Frontend

- **TypeScript:** Strict mode, avoid `any`
- **Components:** Functional with hooks, props via interfaces
- **Naming:** PascalCase for components, camelCase for functions/variables
- **Styling:** Tailwind 4 utility classes
- **Constants:** Size limits and config in `src/lib/constants.ts`

## Testing

### Go Backend

```bash
cd backend

# Run all tests
go test ./... -count=1

# Run a specific package
go test ./internal/service/... -v

# Regenerate mocks after interface changes
go generate ./...
```

Tests use table-driven patterns with mocks generated by `gomock`.

Key test files:
- `internal/realtime/client_test.go` — Client struct, send channel, rate limiter
- `internal/realtime/hub_test.go`, `room_test.go`, `message_test.go` — WebSocket infrastructure
- `internal/handler/audit_handler_test.go` — SystemStats handler (gomock)
- `internal/handler/backup_handler_test.go` — Backup handler (gomock)
- `internal/handler/health_handler_test.go` — Health, version endpoints
- `internal/handler/log_handler_test.go` — Log query/summary handler
- `internal/middleware/*_test.go` — Brute-force, cache, CSRF middleware
- `internal/repository/*_test.go` — Repository unit + integration tests

### Regenerating Mocks

```bash
go generate ./...
```

### Frontend

```bash
pnpm lint          # ESLint
pnpm typecheck     # TypeScript strict type check (tsc --noEmit)
pnpm build         # Full production build (includes type check)
```

│   ├── cmd/server/           # Entry point
│   ├── internal/
│   │   ├── handler/          # HTTP handlers
│   │   ├── service/          # Business logic
│   │   ├── repository/       # Database queries (pgx)
│   │   ├── storage/          # S3 file operations
│   │   ├── middleware/        # JWT, RBAC, rate limit, CSRF
│   │   ├── router/           # Route registration
│   │   ├── realtime/         # WebSocket: hub, rooms, live cursors
│   │   ├── database/         # Database pool + migrations
│   │   ├── pkg/              # Shared utilities (jwt, cookie, response, validation)
│   │   └── testutil/         # Test helpers, fixtures, mocks
│   └── go.mod
├── src/                      # Next.js frontend
│   ├── pages/                # Pages Router
│   ├── components/           # UI components
│   ├── contexts/             # Auth + App state
│   ├── services/             # API + WebSocket + logging clients
│   ├── lib/                  # Constants, hooks, utils
│   └── types/                # Shared TypeScript types
├── docs/                     # Documentation
│   ├── API.md                # API reference
│   ├── ARCHITECTURE.md       # System design
│   └── DEPLOYMENT.md         # Deployment guide
└── docker-compose.yml
```

fix: prevent duplicate WebSocket connections
docs: update API reference for backup endpoints
test: add board version concurrency tests
refactor: extract cursor batching to separate module
```
